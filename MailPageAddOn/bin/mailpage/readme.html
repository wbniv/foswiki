Getting the script to work.

Changes in the CGI files

  • Please check the first line of all the .CGI files. (#!/usr/local/bin/perl). This should be the place where Perl is located on your server. Ask your webmaster for more information.
  • In all the .CGI files, you can find the following lines (They might look different in the script): 
    use LWP::UserAgent;
$ua = new LWP::UserAgent;
$ua->agent("$ENV{'HTTP_USER_AGENT'}");
use CGI::Carp qw(fatalsToBrowser); # Provides you with fatal error message if they occur.
use CGI qw/:standard/;
require '/home/username/mailpage/mailpage.conf';
require $mtvwebdesignlib;
    These lines are probably the most important.
    Now, what you need to do is this: The 
    require '/home/username/mailpage/mailpage.conf';
    line should point to your config file. Please make sure you use an absolute path (as shown). Ask you webmaster for more information.

Changes in the mailpage.conf file

  • $servicename
    This is the name of the service you are providing. You can leave it the way it is.
  • $servicelocation
    This is the URL to the page where people can find out how to place a link to the script on their page so people surfing on their pages can also email their pages.
  • $mtvwebdesignlib
    This should point to the mtvwebdesign.lib file. Make sure it's the absolute path to the file. Example: "/home/username/mailpage/mtvwebdesign.lib";
  • $privacy_file
    This should point to the privacy.txt file. Make sure it's the absolute path to the file. Example: "/home/username/mailpage/privacy.txt";
  • $use_flock
    Set this to 1 if you are on a UNIX or WindowsNT system. If you don't know what system you are on, just set it to 1.
  • $mailprog
    This should point to the mailprogram on your server. Example: "|/usr/bin/sendmail -t";
    Ask you webmaster. Please note that you have to include the pipe (|) and flags like "-t".
  • $usetoken
    Set to 1 if you want to use the tokenfile option. Read more about it here.
  • $tokenfile
    The name of the file containing the $token. (See the example file tokefile.html)
  • $token
    This is the commentline which the script will look for in the $tokenfile. Default is <-- TOKEN -->
  • 1;
    Just don't delete the 1...

Uploading the files

First: Be sure to have the directory read where you want to place your CGI script. Make sure this directory is CHMOD 777. See the FAQ for more information about CHMOD

UPLOAD IN ASCII FORMAT!

  1. mailpage.cgi
    Upload this file to the directory where you want it. CHMOD the file 755.
  2. mailpage.conf
    Upload this file to the place specified in each .cgi file under "require". CHMOD 777.
  3. mtvwebdesign.lib
    Upload this file to the place specified in $mtvwebdesignlib. CHMOD the file 777.
  4. tokenfile.html
    Upload this file to the place specified in $tokefile. CHMOD 644. (Just upload.)

Running the script

You simpley add a link to the script from the pages you want to be available for email: 
<a href="/cgi-bin/mailpage/mailpage.cgi">Email this page!</a>
You can also email other pages: 
<a href="/cgi-bin/mailpage/mailpage.cgi?url=http://mtvwebdesign.hypermart.net">Email the MTVWebdesign page to a friend!</a>
See the test.html file in the package.

FAQ

What is CHMOD?
CHMOD is changing the attributes of a file. (UNIX systems only.). You should do that when you have uploaded the file, or create a new directory. Most FTP clients (like WS_FTP and BulletProofFTP) have the feature, only they call it CHANGE ATTRIBUTES and won't really show number but instead they show READABLE, WRITEBLE and EXECUTABLE. You can find out what you need to do by using this:

BinaryDecimal
000 = 0
001 = 1
010 = 2
011 = 3
100 = 4
101 = 5
110 = 6
111 = 7

r = readable
w = writeble
x = executable

So to set the permission above, "rw-r--r-x", on the file "test.txt," you would figure out the decimal first:
rw- r-- r-x =
110 100 101 =
6 4 5

I get a 501 Internal server error. What should I do?
Well, you have a problem there....
What you should do first is checking if you uploaded the CGI file in ASCII format and not in BINARY format.
If that doesn't work, check if all the permissions (CHMOD) are set right.
If that doesn't work, check the server error log. Ask your webmaster for more information

What does "premature end of script headers mean"?
You probably uploaded the file in BINARY format. This should be ASCII format.
If that doesn't work, you should check the things you have changed in the script:
Is there a ";" at the end of lines? Are there any missing quotes (")? Etceteras.

How do I use a tokenfile?
First, take a look in the source of tokenfile.html. This is a page which has a tag in the source, which you cannot see when you view it in the browser. The tag is <!-- TOKEN --> (specified in setup.txt at $token). When you select that you want to use a template, Chat with me! will look for the $tokenfile (tokenfile.html) and that it 'll search for the $token. If it finds the token (WHICH NEEDS TO BE PLACE ON A SEPARATE LINE IN THE SOURCE OF THE HTML DOCUMENT!), it'll print it's output at that spot in the html document. This way you get your own nice layout for the Chat with me! service!
TIP: Try to use STYLE tags in the $tokenfile!
